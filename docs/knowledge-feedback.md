# 人工审核结果回流知识库

第 11 步把已完成人工审核的分类结果转换为 RAG 可检索的 `approved_example`，形成从模型预测、人工修正到知识复用的数据闭环。

## 准入规则

只有以下结果允许回流：

- `human_approved`：人工确认模型原始标签；
- `human_corrected`：人工提交了修正后的最终标签。

以下结果禁止回流：

- `auto_accepted`：高置信度仍然只是模型预测；
- `review_pending`：尚未完成人工审核；
- 数据不完整、没有审核人或没有最终标签的异常记录。

这一限制可以避免模型自己生成错误标签、又把错误标签召回给下一次分类所形成的知识污染和错误放大。

## 幂等设计

每个回流文档使用 `review-example:{review_task_id}` 作为业务 ID，Qdrant Point ID 由该业务 ID 确定性生成。同一个审核任务重复执行回流时会覆盖原向量点，文档数量不会增加。

全量 `build` 会同时加载静态标签、仓库样例和数据库人工审核样本，因此重新构建 Collection 不会丢失已经审核的动态样本。

## 执行

```bash
creativebench-knowledge feedback
```

如果还没有人工审核完成的任务，命令会输出“没有可回流的人工审核样本”，不会计算 Embedding 或修改向量库。
